Email Ingestion (IMAP)

Email ingestion

Configure a dedicated inbox used to update submission statuses from employer replies. You can run it manually from the Email Ingestion Log, or enable background polling.

{{ form.email_ingest_enabled }}

When turned on, the platform will use the IMAP settings below to connect to this mailbox (once the background worker feature is enabled in a future release).

{{ form.email_auto_poll_enabled }}

When enabled, a background scheduler polls the IMAP inbox automatically. Keep this off if you only want manual runs.

{{ form.email_imap_host }}

e.g. imap.gmail.com

{{ form.email_imap_port }}

993 for SSL/TLS is recommended.

{{ form.email_imap_use_ssl }}

Keep this on for Gmail and most providers.

{{ form.email_imap_username }}

Usually the full email address of the inbox.

{{ form.email_imap_password }}

Stored encrypted. Leave blank to keep the existing secret.

{{ form.email_poll_interval_seconds }}

How often the background worker should check for new emails (used by the scheduler; also used as a rate-limit guard).

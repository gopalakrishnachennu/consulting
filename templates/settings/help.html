1. IMAP Email Ingestion (Gmail)

This feature reads employer replies from a dedicated inbox and updates submission statuses automatically. Use it with a dedicated Gmail account and an App Password.

1.1 Create / prepare the Gmail inbox

  1. Create or choose a Gmail account, e.g. yourplatform.updates@gmail.com.
  2. Sign in to that account and open https://myaccount.google.com/security.
  3. Turn 2‑Step Verification ON (if not already).
  4. On the 2‑Step Verification page, scroll down and open App passwords (or visit https://myaccount.google.com/apppasswords).
  5. Create an App Password:
    • App: Mail (or “Other (Custom)” → type CHENN IMAP).
    • Generate and copy the 16‑character password (e.g. abcd efgh ijkl mnop).
  6. In Gmail settings → See all settings → Forwarding and POP/IMAP:
    • Enable IMAP access.
    • Save changes.

1.2 Configure CHENN IMAP settings

  1. Go to Settings → Platform Config → Email Ingestion (IMAP).
  2. Turn ON Enable IMAP email ingestion.
  3. Fill the fields:
    • IMAP Host: imap.gmail.com
    • IMAP Port: 993
    • Use SSL/TLS: checked
    • IMAP Username: the same Gmail address that generated the App Password
    • IMAP Password / App Password: paste the 16‑character App Password from Google
    • Poll Interval: keep 60 seconds for most setups.
  4. Optionally enable:
    • Background polling (Celery Beat) – for automatic, periodic polling.
    • AI fallback – uses your existing LLM config when rules are unsure.
    • Email notifications – notify employees / consultants on auto-updates.
  5. Click Save Configuration.

1.3 Test the connection

  1. Go to Applications → Job Applications and click Email Ingestion Log in the top-right.
  2. Click Run IMAP poll now.
  3. Interpret the banner:
    • Success banner: processed n messages → setup is correct.
    • “auth_failed: Application-specific password required”: use a Gmail App Password (normal passwords will fail).
    • “auth_failed: Invalid credentials”: check that IMAP Username matches the account that generated the App Password.
    • “connect_failed”: check host/port, local network, or firewall.

1.4 How employees should use it

  1. Employer replies to an application (e.g. “We’d like to interview John”).
  2. Employee forwards that email to the dedicated inbox (e.g. yourplatform.updates@gmail.com).
  3. Within ~60 seconds, CHENN:
    • Reads the email via IMAP.
    • Uses rules (and optional AI) to detect status.
    • Matches the correct submission when safe.
    • Auto-updates the status and logs it in the timeline.
  4. Employees/admins can review everything in the Email Ingestion Log.

2. LLM Configuration (OpenAI)

CHENN uses OpenAI for resume generation and some analytics features. All usage is tracked and capped.

2.1 Add / rotate the API key

  1. Go to Settings → System Settings → LLM Settings.
  2. Paste your OpenAI API key in the API key field.
  3. Click Save. The key is stored encrypted using the same mechanism as IMAP.
  4. Optionally click Test key to verify that the key is valid and list available models.

2.2 Control cost and safety

  • Temperature – higher = more creative, lower = more deterministic.
  • Max output tokens – upper bound on response length.
  • Monthly token cap – hard cap; when reached, generation can auto-disable.
  • Generation enabled – master toggle to quickly pause all LLM calls.

All LLM usage is logged in LLM Logs with token counts and cost estimates so you can audit and tune usage over time.

3. Background Workers (Celery)

Celery workers let CHENN run tasks like IMAP polling in the background.

3.1 Services you should run

  • Redis – message broker / result backend.
  • Celery worker – executes tasks such as poll_email_ingest_task.
  • Celery Beat – scheduler that triggers periodic tasks.

3.2 How IMAP polling uses Celery Beat

  • When you save PlatformConfig with:
    • Enable IMAP email ingestion = ON
    • Enable background polling = ON
  • CHENN creates or updates a periodic task:
    • Name: CHENN: IMAP email ingestion poller
    • Task: core.tasks.poll_email_ingest_task
    • Interval: email_poll_interval_seconds (minimum 30s).
  • As long as Celery Beat and a worker are running, IMAP will be polled automatically at that interval.
  • You can still run manual polls from the Email Ingestion Log at any time.

Quick IMAP checklist

  • 2‑Step Verification turned ON.
  • App Password generated for Mail.
  • IMAP enabled in Gmail settings.
  • imap.gmail.com:993 with SSL.
  • Username matches the account that created the App Password.

Troubleshooting banners

  • auth_failed: Application-specific password required – use a Gmail App Password, not your normal password.
  • auth_failed: Invalid credentials – username and App Password do not belong to the same account.
  • connect_failed – check IMAP host / port 993 / local firewall.
  • search_failed – IMAP search failed; try again or check inbox via Gmail UI.
